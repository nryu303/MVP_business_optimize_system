# 操作マニュアル

営業支援システム MVP — 運用者向け手順書。

## 0. 画面構成

サイドバーは MIKOMERU 準拠の 3 グループ構造:

```
■ トップレベル
  ├─ ホーム           → ダッシュボード (KPI・グラフ・最新ジョブ)
  └─ 案件             → 案件 CRUD / 紐付け管理

■ フォーム送信
  ├─ 自動送信         → ジョブ一覧 / 新規ジョブ / 進捗
  ├─ 自動送信ログ     → 配信結果一覧 / CSV エクスポート
  ├─ 送信文章テンプレート → 件名・本文・変数
  ├─ 送信元テンプレート → 差出人プロファイル
  └─ 送信除外設定     → ブラックリスト管理

■ 会社情報
  ├─ リスト取込       → CSV アップロード
  └─ 保存済みリスト   → リスト / 企業 CRUD

■ その他
  └─ 設定             → アカウント情報 / 文字数プリセット / 既定値
```

## 1. 通し運用シナリオ (初回利用)

### 1-1. ログイン
http://localhost:3002/login

- Email: `admin@example.com`
- Password: `admin1234`

初回セットアップ直後は上記デフォルト値。変更は `.env` の `SEED_ADMIN_PASSWORD` を設定し直して再 seed、または DB 上のパスワードハッシュを直接更新。

### 1-2. 送信元テンプレートの登録
**サイドバー「送信元テンプレート」→ ＋ 新規作成**

差出人プロファイル (会社名・担当者氏名・メール・電話・住所 等) を登録しておくと、送信ジョブ作成時に選ぶだけで各フォームに自動入力されます。

- 必須: テンプレート名 / 会社名 / 担当者氏名 / メール
- 複数登録可 (キャンペーン別・部署別などで使い分け)

### 1-3. 送信文章テンプレートの登録
**サイドバー「送信文章テンプレート」→ ＋ 新規作成**

- 件名 + 本文で構成
- 変数 `{{会社名}}` `{{担当者名}}` が使用可 (送信時に各社の名前に置換)
- 右側プレビューで変換結果を常時確認可能
- 「文字数プリセット」で上限目安を設定 (超過は赤字表示)
- 「複製」ボタンで類似テンプレを量産可

### 1-4. ブラックリストの登録
**サイドバー「送信除外設定」**

不要先を事前登録しておくと、CSV 取込時 + 送信直前の 2 段階で自動除外されます。

- **種別 = ドメイン** — `example.com` を登録 → そのドメイン全サブドメインを除外
- **種別 = 会社名** — 完全一致で除外
- CSV 取込時に「営業お断り」「営業不要」等のキーワードを含む会社名は自動的に BL 追加 (種別=会社名・登録元=取込時自動)

### 1-5. 営業先リストの取込 (CSV)
**サイドバー「リスト取込」**

#### 準備する CSV (UTF-8 または Shift_JIS)
```csv
会社名,問い合わせフォームURL,サイトURL,メールアドレス,業種
株式会社A,https://a.example.com/contact,https://a.example.com,info@a.example.com,IT
株式会社B,https://b.example.com/inquiry,,,広告
```

- **必須項目**: 会社名 / 問い合わせフォームURL
- **任意項目**: サイトURL / メールアドレス / 業種
- ヘッダ名は自動検出されます (英語・日本語両対応)

#### 手順
1. CSV を選択 → 「解析」ボタン
2. 項目マッピングを確認・調整 (通常は自動検出でOK)
3. リスト名を入力 → 「取込実行」
4. 結果カードで `取込成功 / 重複除外 / BL除外` を確認
5. バリデーションエラーがあれば行番号とメッセージが表示される

### 1-6. リストの確認・編集
**サイドバー「保存済みリスト」**

- リスト一覧で企業数・紐付け案件数を確認
- 各リストをクリック → リスト名変更 / 企業の手動追加 / 企業個別編集・削除

### 1-7. 案件作成と紐付け
**サイドバー「案件」→ ＋ 新規案件**

- 案件名 / スポンサー / ジャンル / 期間 / ステータスを設定
- 作成後、案件詳細ページで **リスト** と **送信文章テンプレ (+送信元テンプレ)** を紐付け
- 紐付け済みリスト・テンプレは送信ジョブ作成時に優先候補として表示される

### 1-8. 自動送信ジョブの作成・実行
**サイドバー「自動送信」→ ＋ 新規ジョブ**

4 ステップのウィザード:
1. 案件を選択
2. リストを選択 (紐付け済みが優先表示)
3. 送信文章テンプレ + 送信元テンプレ (任意) を選択
4. 確認 → 「承認して実行」

制約:
- 1 ジョブあたりの上限: **50 件** (超過時はエラー表示。分割して実行)
- 同一ドメイン連続送信時は 2 秒間隔
- 各社への送信失敗時は最大 2 回リトライ (合計 3 試行)

### 1-9. 進捗確認と制御
ジョブ詳細画面は **5 秒ごとに自動更新**。

- 各社の状態が `待機 → 送信中 → 成功 / 失敗 / スキップ` と遷移
- **一時停止** → 現在処理中の 1 件を終えてから停止
- **再開** → キューに再登録、未処理分を続行
- **キャンセル** → 以降処理せず終了

## 2. 結果の確認・分析

### 2-1. ダッシュボード (ホーム)
**サイドバー「ホーム」**

- **KPI カード**: 本日送信 / 本日成功率 / 今月送信 / 今月成功率 / 進行中案件 / 今月新規案件
- **日次棒グラフ** (直近 14 日): 成功・失敗・スキップの内訳
- **案件別棒グラフ** (上位 8 案件): 案件ごとの送信実績
- **最新ジョブログ** (直近 10 件)

### 2-2. 配信結果一覧 (P12)
**サイドバー「自動送信ログ」**

- **件数カード** (4 分類): 成功 / 失敗 / 営業拒否 / フォームなし
- **絞込**: 案件 / 分類 / 実行日範囲
- **ページング**: 50 件/ページ
- 各行から「ジョブへ」でジョブ詳細に遷移可能

### 2-3. エラー種別 5 分類

| 分類 | 意味 | 典型パターン |
|---|---|---|
| **成功** | 送信成立 | 送信完了ページを検出 |
| **失敗** | ネットワーク / 検証エラー | タイムアウト・HTTP エラー・バリデーション失敗 |
| **営業拒否** | ブラックリスト除外 | 事前 BL 登録先・自動キーワード検出先 |
| **フォームなし** | フォーム構造の自動検出不可 | 対象サイトに `<form>` が無い or 非標準構造 |
| **キャンセル** | 手動中止 | ジョブ全体レベル |

### 2-4. CSV エクスポート
配信結果一覧画面右上「CSV エクスポート」で絞込条件付きのまま CSV ダウンロード可能。UTF-8 BOM 付き (Excel 直接開き対応)。

## 3. 設定

**サイドバー「設定」**

- **アカウント情報**: 表示のみ (編集は DB 直接)
- **文字数プリセット**: 送信文章テンプレ編集画面で表示する文字数ボタン (例: `200, 500, 1000`)
- **送信元テンプレート既定値**: 送信ジョブ作成時に事前選択される送信元

## 4. トラブルシューティング

### 送信ジョブが「待機中」のまま進まない
→ worker プロセスが起動していない可能性。`npm run dev:worker` のターミナルを確認。

### 特定のサイトで「フォームなし」エラーが多発
→ 対象サイトのフォーム構造が非標準。MVP スコープ外のケース (個別に form_url 確認 or サポート外として BL 登録)

### 送信数が想定より少ない
→ BL 除外件数と「重複除外」件数を取込結果 / ジョブ詳細で確認。同一ドメインは 1 件に集約される仕様 (MVP 範囲)。

### パスワードを忘れた
→ DB 側で直接更新:
```bash
node -e "require('bcryptjs').hash('newpass', 10).then(console.log)"
# 出力ハッシュを:
# UPDATE users SET password_hash='...' WHERE email='admin@example.com';
```

## 5. 定期運用の目安

| 頻度 | 作業 |
|---|---|
| 送信前 | ブラックリスト更新・新規リスト取込 |
| 送信実行 | 承認ウィザード → 進捗監視 (1 ジョブ ~15 分) |
| 日次 | ダッシュボード確認・失敗率異常チェック |
| 週次 | CSV エクスポートで集計 / 取引先反応の突合 |
| 案件終了時 | 案件ステータス変更 (配信中 → 完了) |
